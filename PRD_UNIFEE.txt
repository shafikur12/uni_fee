================================================================================
1. EXECUTIVE SUMMARY
================================================================================

UniFee is a comprehensive digital platform designed to streamline university fee 
management and exam permission slip distribution. The system facilitates seamless 
communication between students, administrative staff, and accountants, ensuring 
transparent fee tracking, instant exam eligibility verification, and automated 
permission slip generation.

Key Value Propositions:
- Eliminates manual fee collection processes
- Provides real-time payment status and exam eligibility tracking
- Automates exam permission slip approval workflow
- Reduces administrative overhead and paperwork
- Enhances transparency and accountability

================================================================================
2. PRODUCT VISION
================================================================================

"Empower universities to manage student fees efficiently and fairly, enabling 
instant exam permission while maintaining complete financial transparency."

Target Users:
- Students: Submit fees and receive instant exam permission slips
- Accountants: Track payments and verify fee submissions
- Registrars/Administrators: Manage exam permissions and system configuration
- System Administrators: Oversee platform operations and staff accounts

3.1 AUTHENTICATION & USER MANAGEMENT
────────────────────────────────────

3.1.1 Student Authentication
- Sign up: Email, password, full name, student ID, batch code
- Email verification (optional, can be disabled for faster access)
- Login with email/password
- Session management with Supabase Auth
- Password reset functionality
- Account security (bcrypt password hashing)

3.1.2 Staff Authentication
- Admin staff created manually (no public signup)
- Three roles: Admin, Accountant, Registrar
- Login with email/password
- Custom staff profile creation
- Role-based access control (RBAC)
- Persistent sessions with secure cookies

3.1.3 Database Schema
Students Table (auth.users):
- id (UUID, primary key)
- email (unique)
- encrypted_password
- created_at
- updated_at

Student Profiles Table:
- id (UUID, foreign key to auth.users)
- full_name
- student_id (roll number)
- batch_id (references batches table)
- email (reference)
- created_at
- updated_at

Staff Profiles Table:
- id (UUID, foreign key to auth.users)
- role (Admin, Accountant, Registrar)
- department
- assigned_batches (JSON array)
- created_at
- updated_at

Batches Table:
- id (UUID, primary key)
- batch_code (e.g., "BATCH-2024-01")
- batch_name
- status (Active, Inactive)
- created_at

3.5 WORKFLOW & PROCESS FLOW
───────────────────────────

Student Journey:
1. Student registers with email, password, name, student ID, batch code
2. Email verification (optional)
3. Redirected to student dashboard
4. Uploads payment receipt
5. Accountant reviews and approves fee
6. Student notified of approval
7. Exam permission automatically generated
8. Student downloads permission slip

Staff Workflow:
1. Admin creates staff accounts (manual process)
2. Staff login redirected to appropriate dashboard based on role
3. Accountants review receipts → Approve/Reject
4. Registrars process exam permissions → Generate slips
5. System generates audit logs
